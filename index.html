Introduction

N2O defines a way to scale applications from protocols to modules, events persistence and schema. It replaces CORBA/SOAP/XMPP in small and compact binary manner, yet it's ready for high-speed, low-latency IoT, MQTT, and WebSocket applications. You can embed it into cowboy, emqttd or any message processing loop.

N2O Applications

N2O Applications is the family of services that rely on N2O as a core router and processing loop. We've built banks, games and messaging apps in a form of N2O protocols.

N2O itself provides robust infrastructure for sessions, cookies, auth, message queue, controls, rest services, javascript compiler and many more. Limited in size to 1K LOC.

The packages are available through Github and Hex repositories and are compatible with mad and rebar package managers. We have manpage manuals, PDF/HTML generated with LaTeX and HeVeA and number of turtorials, just search "n2o erlang web". N2O is 4 y.o. has 20 releases and 40 contributors.

N2O Protocols are sets (or sums) of messages that could be handled within single module. You can plug as many unintersected protocols to application as you want. N2O performs calls to servers started from protocol code. The worker pool is an adjustable hash ring. In MQTT version workers are regular MQTT clients.
