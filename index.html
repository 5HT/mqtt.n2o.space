N2O ECO

SYNOPSIS

N2O ECO defines a full stack specification along with reference implementations for functional languages. This language agnostic stack is a story of functional programming and communication technologies manifested as a series of minimalistic applications and extensions to the base library of implementation language.

AUTHOR'S GUIDE

Each language, N2O is implemented in, should embed N2O most naturally and compact. If some layer between the base library of a language is needed, it could be provided, but it should be reduced to zero if possible. In some cases, some parts of a library could be replaced with own replacement. N2O should provide client companion library usually implemented in a different set (than server's stack) of client languages (while other prefer monolingual stack (often homoiconic languages) rather that multilingual). The reason is that for a mobile client's native languages and environment are slightly different and the aim is to provide the fastest and most native way possible with hand-coded or partially generated client's stack. N2O should provide by design the ability to mount, intercept, inject and embed any lambda at any mount point. Don't take this as an extreme. If you did everything right, N2O should not be more than 500 LOC in any language. As an example, three N2O implementations presented here in three clean functional languages: Standard ML, Haskell, and Erlang.

O1 ECO

O1 ECO is a Standard ML N2O implementation. Standard ML is a certified language with formally defined operational semantics. Here is yet simple CML loop with INetSock.TCP spawned acceptors. Simple enough to handle N2O protocols inside the TCP server.

  • o1/n2o — N2O 0.11 ML

O3 ECO

O3 ECO introduces N2O protocols for Haskell programming language. The aim is to be compatible at transport level with Erlang version of N2O. Here is full implementation of Nitrogen Web Framework in Haskell along with simple yet powerful N2O core and HTML5 example driven by WebSocket and static HTTP servers.

  • o3/n2o — N2O 0.11 HS
  • o3/nitro — NITRO 0.11
  • o3/sample — SAMPLE 0.11

O7 ECO

O7 ECO defines ecosystem, a full-stack specification and implementation for banking, messaging and platform development. It specifies the way to scale applications, protocols, processes, services, and storage. Ready for high-speed, low-latency MQTT, TCP, QUIC, UDP, and WebSocket applications.
